# 初心者向け：Claude Code でポートフォリオサイトを作る手順マニュアル

このマニュアルは **「Claude Code をインストールしたばかりで、コードも触ったことがない人」** が、参考画像 1 枚からポートフォリオサイトを作るための手順書です。

所要時間: **30 分〜1 時間**（写真の用意込み）

---

## このマニュアルでできるようになること

- 参考画像を見せて「これと同じデザインで作って」と Claude に指示
- 自分の写真と実績画像を組み込む
- 配色や余白を細かく調整
- ブラウザで動くサイトをローカルで表示

---

## STEP 0. 事前準備（一度だけやればOK）

### 0-1. Claude Code が動く状態になっているか確認

ターミナル（Mac なら「ターミナル.app」、Windows なら「PowerShell」または「Windows Terminal」）を開いて、以下のコマンドを実行:

```bash
claude --version
```

バージョン番号が表示されれば OK。されない場合は Claude Code 公式の手順でインストールし直してください。

### 0-2. Python が入っているか確認（プレビュー表示に使います）

```bash
python3 --version
```

`Python 3.x.x` のような表示が出れば OK。**Mac には標準で入っています**。Windows で入っていない場合は <https://www.python.org/downloads/> からインストール。

---

## STEP 1. 作業フォルダを作る

サイトのファイルを置く専用のフォルダを作ります。

### Mac の場合

```bash
mkdir -p ~/Desktop/ポートフォリオ
cd ~/Desktop/ポートフォリオ
```

`~/Desktop/` は「デスクトップ」のことです。デスクトップに「ポートフォリオ」というフォルダができます。

### Windows の場合

```powershell
mkdir $HOME\Desktop\ポートフォリオ
cd $HOME\Desktop\ポートフォリオ
```

> 💡 **`cd` は「そのフォルダの中に移動する」コマンド**です。以降、ターミナルでの作業はこのフォルダ内で行われます。

---

## STEP 2. このフォルダの中で Claude Code を起動

ターミナルでフォルダに入っている状態のまま、以下を実行:

```bash
claude
```

Claude Code が起動し、入力待ちの状態になります。これ以降は **Claude にメッセージを送るだけ** で作業が進みます。

---

## STEP 3. 参考画像を探す

「こんな感じのサイトが作りたい」というイメージ画像を 1 枚用意します。

### おすすめの探し場所

- **Pinterest** （`portfolio website`, `ポートフォリオサイト` などで検索）
- **Awwwards** (<https://www.awwwards.com/>)
- **MUUUUU.org** (<https://muuuuu.org/>) — 日本のサイトギャラリー
- **SANKOU!** (<https://sankou.jp/>) — 日本のWebデザインギャラリー

気に入った画像が見つかったら、**スクリーンショットまたは画像保存**しておきます。

### コツ

- レイアウトがシンプルなものを選ぶと再現精度が上がります
- 1 枚で全体構成（ヒーロー / About / 作品一覧 / フッター）が見えているものがベスト

---

## STEP 4. Claude に参考画像を渡して、最初の指示を出す

Claude Code のチャット欄に **画像をドラッグ&ドロップ**するか、貼り付け（`Cmd+V` / `Ctrl+V`）してから、以下のような指示を送ります。

### 📋 コピペして使えるテンプレート

```
この写真と全く同じポートフォリオサイトを作ってください。
デザインはこれを参考にして、ただし色味だけ少し変更したいので、
〇〇がかった色味にしてください。

ウェブサイト作成のポートフォリオのような形式で、
タブによってバナーやウェブサイトをいろいろと切り替えられるように、
すべて写真通りに設定してローカルホストで表示させてください。
```

`〇〇` には好きな色（例：紫、青、緑、ピンクなど）を入れます。

> 💡 「ローカルホストで表示」と書くと、Claude が自動で `python3 -m http.server` を起動して、ブラウザですぐ確認できる状態にしてくれます。

### Claude がやってくれること

- `index.html` `styles.css` `script.js` の 3 ファイルを作成
- 写真の代わりに**ダミー画像（プレースホルダー）**で一旦組み上げる
- ローカルサーバーを起動して URL を教えてくれる（例: `http://localhost:8765`）

ブラウザでその URL を開いて、参考画像と同じ構成になっているか確認します。

---

## STEP 5. 写真を格納する

Claude が一旦ダミーで作ったサイトに、自分の写真を入れていきます。

### 5-1. 画像を入れるフォルダを作る

Claude に以下のように指示するか、Finder / エクスプローラーで手動で作成:

```
プロジェクトに image というフォルダを作ってください。
さらにその中に「実績」というフォルダを作ってください。
```

これで以下の構成になります:

```
ポートフォリオ/
├── index.html
├── styles.css
├── script.js
└── image/
    └── 実績/
```

### 5-2. 写真を image フォルダにコピーする

**Finder（Mac）/ エクスプローラー（Windows）** で以下のように配置します:

| 写真の用途 | 入れる場所 | ファイル名の例 |
|---|---|---|
| 自分のプロフィール写真（ヒーロー画像） | `image/` 直下 | `profile.jpg` など何でも OK |
| 作品のスクリーンショット（実績） | `image/実績/` 内 | `企業HP.png`, `サロンサイト.png` など分かりやすく |

> 💡 **ファイル名に日本語を使っても問題ありません**。スペースが入っていても OK。

### 5-3. Claude に写真の場所を伝える

```
写真は格納されています。
プロフィール写真は image/profile.jpg です。
こっちに変えて。
```

Claude が `index.html` の画像パスを書き換えてくれます。実績画像も同様に:

```
実績は image/実績/ に格納してあります。
それを使ってください。
```

---

## STEP 6. 細かい調整を指示する

ここからは「気になるところを Claude に伝える」だけで OK。今回の制作で実際に使われた指示の例を紹介します。

### よく使う指示テンプレート集

| 直したいこと | 指示の例 |
|---|---|
| 余白が広すぎる | 「かなりスペースが空いているので、無駄な余白は全部消してください」 |
| 名前を変更 | 「名前は 〇〇 だけにして」 |
| 写真を別のものに | 「写真を image/xxx.jpg に変えてください」 |
| 装飾を追加 | 「もうちょっと図形とかをぷかぷか浮かせて、リッチなサイトに仕上げてください」 |
| 装飾を減らす | 「丸はもっと減らしてください。一つ一つの丸を大きくして数を減らして」 |
| 装飾の種類を変える | 「三角なども入れて幾何学的にしてほしいです。数は多めで」 |
| 文字や写真を大きく | 「文字などをもっと大きくしてください。写真も」 |
| 実績の数を増やす | 「実績は写真を格納してあるのでそれを使ってください。余白を綺麗に埋めて」 |
| 画像の比率を統一 | 「サイズは全部 16:9 で表示させるように」 |

### 上手な伝え方のコツ

- ✅ **「もっと大きく」「もっと少なく」など、相対的な指示で OK**
- ✅ **画面のスクリーンショットを送って「ここがおかしい」と伝える**と精度が上がる
- ✅ 一度に複数の修正を伝えても OK（「余白を詰めて、文字を大きく、写真も大きく」など）

---

## STEP 7. ブラウザで確認する

サーバーは Claude が立ち上げっぱなしにしてくれているので、ファイルを保存するたびに **ブラウザをリロード（`Cmd+R` / `F5`）** すれば最新の状態が見られます。

URL は Claude が教えてくれます（だいたい `http://localhost:8000` または `http://localhost:8765`）。

> 💡 リロードしても変わらないときは、**強制リロード（`Cmd+Shift+R` / `Ctrl+Shift+F5`）** を試してください。

---

## STEP 8. 終わったら（保存・共有・停止）

### サーバーを止める

Claude に「サーバーを止めてください」と伝えるか、ターミナルで `Ctrl + C`。

### 後でまた編集したいとき

1. ターミナルで `cd ~/Desktop/ポートフォリオ` （Windowsは `cd $HOME\Desktop\ポートフォリオ`）
2. `claude` で Claude Code を起動
3. 「前回の続きから〇〇を直して」と指示

ファイルが残っているので Claude は中身を読んで作業を再開できます。

### 友達に見せたい / 公開したい

Claude にこう聞けば手順を教えてくれます:

```
このサイトを無料で公開する方法を教えてください。
Vercel か GitHub Pages を使いたいです。
```

## トラブル別 困ったときの対処

| 症状 | 対処法 |
|---|---|
| `claude` コマンドが効かない | ターミナルを一度閉じてもう一度開く / Claude Code を再インストール |
| 画像が表示されない | ファイル名のスペル・拡張子（`.jpg` / `.JPG`）が一致しているか確認。Claudeに「画像が表示されません」と伝えれば調べてくれる |
| `http://localhost:xxxx` が開かない | ターミナルで `python3 -m http.server 8765` を再実行 |
| 修正が反映されない | 強制リロード（`Cmd+Shift+R` / `Ctrl+Shift+F5`） |
| デザインが崩れた | 「ひとつ前の状態に戻して」と Claude に頼めば、ほぼ戻せます |

## 今回 Claude が裏側で使った技術（参考）

「中身が気になる人向け」のおまけ情報です。何も知らなくても作れます。

| 技術 | 使い道 |
|---|---|
| **HTML5** | サイトの骨組み |
| **CSS3** | 見た目・配色・アニメーション・浮遊シェイプ |
| **JavaScript** | タブの切り替え機能 |
| **Google Fonts** | 英語フォント・日本語フォント |
| **Python 簡易サーバー** | ブラウザで確認するためのローカル配信 |

フレームワーク（React、Vue 等）や有料ツールは一切使っていません。**HTML / CSS / JS の 3 ファイルだけ**でできています。

---

## このマニュアルの使い方まとめ

```
1. STEP 0 で環境チェック（初回のみ）
       ↓
2. STEP 1〜2 でフォルダを作って Claude を起動
       ↓
3. STEP 3 で参考画像を探す
       ↓
4. STEP 4 で画像を Claude に貼り付けて指示を出す
       ↓
5. STEP 5 で自分の写真を格納
       ↓
6. STEP 6 で気になるところを修正
       ↓
7. STEP 7 でブラウザ確認 → STEP 6 に戻る（納得するまで繰り返し）
       ↓
8. STEP 8 で完了
```

---

**完成形のサンプル** がこのフォルダ内にあります（`index.html` `styles.css` `script.js`）。コードを直接編集する必要はなく、すべて Claude に日本語で伝えるだけで OK です。
